# Changelog

本文件记录 FarmTogether2.AutoSellMod 的版本变化。

## [1.1.3] - 2026-07-15

### 变更

- 移除联网模式每次只能出售一份交易量的人为限制；在线和离线模式现在都会按超出目标的数量批量出售，并继续遵守商店剩余次数和原生 `32767` 笔传输边界。
- 运行时指纹不匹配改为告警而非停用整个插件；基于当前游戏 API 的出售逻辑会继续加载。

## [1.1.2] - 2026-07-15

### 变更

- 已验证并支持 Steam build `24184988`，同步更新游戏二进制指纹。

## [1.1.1] - 2026-07-14

### 修复

- 完整权限玩家不再调用可能失效的原生商店开放状态检查。
- 权限受限时保留原生检查结果；检查抛出异常时跳过对应商店。异常警告使用统一的 10 秒节流窗口，写入时包含城镇槽位与完整异常信息。
- 联网出售改为同一资源一次只保留一个待确认请求，每次请求只执行一份交易量。插件在原生调用前保存 session、资源数量及 Coins、Bills、Medals；只有当前调用内同步出现的完整匹配事件，加上原生调用正常返回，才会记录出售日志和显示提示。
- dispatch 前重新读取当前报价，并调用游戏自己的 `FarmMoney * float` operator 计算三种收益币种；确认签名与游戏的 `float` 精度和取整语义保持一致。
- 仓库虽然已满但资源总量不足一份商店交易时，不再强制发送必然失败的出售请求。
- 待确认请求在 15 秒后进入结果未知状态。该资源不会自动重试；晚到事件和手动交换不会解除阻止状态，只有农场、当前玩家或插件生命周期重置才会清除请求。
- 原生出售调用抛出异常时丢弃本次同步事件记录，同时保留待确认状态，避免无法确定副作用的请求与后续报价重叠。
- session identity 使用 Stage、Farm 和 LocalPlayer 的原生指针，并加入 Stage 与 LocalPlayer 的 Unity instance ID。interop 生成新 managed wrapper 时不会误判为 session 切换；identity 暂时不可读时也不会清除结果未知的请求。
- `Update`、`OnGUI` 和城镇事件回调覆盖完整异常边界，异常日志写入失败时不会继续抛出。
- 插件保存并独占运行时组件；卸载时先停用回调，再清除待确认状态、取消事件订阅并销毁组件。重复加载不会创建第二个出售组件。
- 原生交易次数限制为 16 位有符号整数的正数范围，避免游戏把 `32768` 以上的编码值解读为负数。
- 加载时校验 `GameAssembly.dll` 与 `global-metadata.dat` 指纹；未知游戏二进制不会启用出售逻辑。

## [1.1.0] - 2026-07-10

### 新增

- 默认允许通过 Event Shack 出售 `Event` 与 `EventB` 资源。
- 报价按奖章、钻石、金币的顺序处理；含多种收益币种的报价按其中最高优先级币种排序，相同优先级保持发现顺序。
- 候选报价按排序依次重新评估。执行每项报价前都会重读库存和剩余使用次数，并根据该报价的单次交易量重新计算可售数量。
- 每项报价分别读取和执行，单项失败不再中止同轮其余报价。

### 变更

- `ExcludedResources` 默认值改为 `GoldNugget`。
- 旧默认排除项只迁移一次；自定义值和迁移完成后的修改保持不变。

## [1.0.0] - 2026-06-13

### 新增

- 按仓库存量比例和商店交易量自动出售资源。
- 支持资源排除项、仓库满额时单次出售、出售提示和调试日志。
